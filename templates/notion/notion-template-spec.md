## 系統總覽

### 設計理念

這套 Notion 工作系統是為了讓房仲把 AI 的產出「落地」到日常工作流程中。
光會用 AI 生成內容還不夠，你需要一個地方**收集、管理、追蹤、再利用**這些內容。

### 五大資料庫

| # | 資料庫名稱 | 核心功能 | 對應課程模組 |
|---|-----------|---------|-------------|
| 1 | **客戶 CRM** | 客戶資訊管理 + 溫度分級 + 沉默天數追蹤 | M4 AI 追蹤成交系統 |
| 2 | **物件素材庫** | 物件資訊 + 跨平台行銷進度追蹤 | M3 AI 物件行銷工廠 |
| 3 | **跟進行事曆** | 自動提醒 Day3/7/14/30/90 跟進時機 | M4 AI 追蹤成交系統 |
| 4 | **社群內容排程** | 多平台素材管理 + 排程發布追蹤 | M5 AI 公域獲客飛輪 |
| 5 | **成交客戶關懷** | 4 節點關懷追蹤 + 轉介紹管理 | M4 AI 追蹤成交系統 |

### 1.3 公式設定

#### 沉默天數（Formula）

**欄位名稱**：`沉默天數`
**類型**：Formula
**公式**：

```
dateBetween(now(), prop("最後互動日"), "days")
```

**說明**：計算從「最後互動日」到今天經過了幾天。數字越大，代表越久沒聯繫。

**進階版公式（含風險提示）**：

```
if(
  empty(prop("最後互動日")),
  "⚠️ 未記錄",
  if(
    dateBetween(now(), prop("最後互動日"), "days") >= 30,
    "🚨 " + format(dateBetween(now(), prop("最後互動日"), "days")) + " 天",
    if(
      dateBetween(now(), prop("最後互動日"), "days") >= 14,
      "⚠️ " + format(dateBetween(now(), prop("最後互動日"), "days")) + " 天",
      if(
        dateBetween(now(), prop("最後互動日"), "days") >= 7,
        "🟡 " + format(dateBetween(now(), prop("最後互動日"), "days")) + " 天",
        "✅ " + format(dateBetween(now(), prop("最後互動日"), "days")) + " 天"
      )
    )
  )
)
```

**顯示效果**：
- ✅ 3 天（正常範圍）
- 🟡 8 天（該跟進了）
- ⚠️ 16 天（快流失了）
- 🚨 35 天（緊急喚醒）

### 1.4 視圖設計

#### 視圖 1：溫度看板（Board View）— 主視圖

**設定方式**：
- **視圖類型**：Board（看板）
- **分組依據**：客戶溫度
- **欄位排列**：A-熱 → B-溫 → C-冷 → D-沉睡
- **卡片顯示屬性**：
  - 客戶姓名（Title）
  - 需求區域
  - 預算範圍
  - 沉默天數
  - 帶看次數
- **排序**：沉默天數 — Descending（沉默最久的排最上面）
- **篩選**：無（顯示全部）

**使用情境**：每天早上打開，一眼看出哪些客戶該優先聯繫。從右邊（D-沉睡）拖到左邊（A-熱），視覺化客戶升溫過程。

#### 視圖 2：全部客戶列表（Table View）

**設定方式**：
- **視圖類型**：Table（表格）
- **顯示欄位**：
  1. 客戶姓名
  2. 客戶溫度
  3. 客戶類型
  4. 需求區域
  5. 預算範圍
  6. 沉默天數
  7. 帶看次數
  8. 最後互動日
  9. 來源管道
  10. 負責人
- **排序**：沉默天數 — Descending
- **篩選**：無

**使用情境**：需要搜尋特定客戶、批量檢視或匯出資料時使用。

#### 視圖 3：互動日曆（Calendar View）

**設定方式**：
- **視圖類型**：Calendar（日曆）
- **日期欄位**：最後互動日
- **卡片顯示**：客戶姓名、客戶溫度

**使用情境**：快速看出哪些日子特別忙、哪些日子沒有互動記錄。搭配跟進行事曆一起看，確保沒有遺漏。

#### 視圖 4：本週需跟進（Table View + Filter）

**設定方式**：
- **視圖類型**：Table
- **篩選條件**：
  - 沉默天數 ≥ 7
  - AND 客戶溫度 ≠ D-沉睡
- **排序**：客戶溫度 — Ascending（A-熱排最前）

**使用情境**：每週一早上看一次，找出本週必須跟進的客戶。

#### 視圖 5：熱客追蹤（Table View + Filter）

**設定方式**：
- **視圖類型**：Table
- **篩選條件**：客戶溫度 = A-熱
- **排序**：沉默天數 — Descending

**使用情境**：只看 A 級熱客，集中火力追蹤最有機會成交的客戶。

#### 視圖 6：首購族清單（Table View + Filter）

**設定方式**：
- **視圖類型**：Table
- **篩選條件**：客戶類型 = 首購族
- **排序**：建立日期 — Descending

**使用情境**：首購族通常需要最多教育和引導，這個視圖幫你集中管理。

### 2.3 公式設定

#### 單價（Formula）

**欄位名稱**：`單價`
**公式**：

```
if(
  prop("坪數") > 0,
  round(prop("總價") / prop("坪數") * 100) / 100,
  0
)
```

**說明**：總價（萬）除以坪數，自動算出每坪單價（萬/坪），四捨五入到小數第 2 位。

**顯示格式**：數字 + 「萬/坪」

#### 行銷完成度（Formula）

**欄位名稱**：`行銷完成度`
**公式**：

```
round(
  (
    (if(prop("591完成"), 1, 0) +
     if(prop("LINE完成"), 1, 0) +
     if(prop("FB完成"), 1, 0) +
     if(prop("IG完成"), 1, 0) +
     if(prop("YT完成"), 1, 0) +
     if(prop("Google完成"), 1, 0))
    / 6
  ) * 100
)
```

**說明**：計算 6 個行銷平台 checkbox 的完成百分比。全部打勾 = 100%。

### 5.3 公式設定

#### 關懷進度（Formula）

**欄位名稱**：`關懷進度`
**公式**：

```
round(
  (
    (if(prop("Day0 感謝"), 1, 0) +
     if(prop("Day7 入住關心"), 1, 0) +
     if(prop("Day30 轉介紹"), 1, 0) +
     if(prop("Day90 二次轉介紹"), 1, 0))
    / 4
  ) * 100
)
```

**說明**：計算 4 個關懷節點的完成百分比。全部打勾 = 100%。

**進階版（含視覺指示）**：

```
if(
  and(prop("Day0 感謝"), prop("Day7 入住關心"), prop("Day30 轉介紹"), prop("Day90 二次轉介紹")),
  "✅ 100% 完成",
  if(
    and(prop("Day0 感謝"), prop("Day7 入住關心"), prop("Day30 轉介紹")),
    "🟡 75% — 差 Day90",
    if(
      and(prop("Day0 感謝"), prop("Day7 入住關心")),
      "🟡 50% — 差 Day30",
      if(
        prop("Day0 感謝"),
        "🟠 25% — 差 Day7",
        "🔴 0% — 尚未開始"
      )
    )
  )
)
```

**顯示效果**：
- 🔴 0% — 尚未開始
- 🟠 25% — 差 Day7
- 🟡 50% — 差 Day30
- 🟡 75% — 差 Day90
- ✅ 100% 完成

#### 成交周年（Formula）

**欄位名稱**：`成交周年`
**公式**：

```
if(
  empty(prop("成交日期")),
  "",
  if(
    dateBetween(now(), prop("成交日期"), "years") == 0,
    "尚未滿 1 年（" + format(dateBetween(now(), prop("成交日期"), "days")) + " 天）",
    "第 " + format(dateBetween(now(), prop("成交日期"), "years")) + " 年"
  )
)
```

**說明**：自動算出成交幾周年。滿周年時可以發一則祝賀訊息，維繫關係。

**進階版（含周年提醒）**：

```
if(
  empty(prop("成交日期")),
  "",
  if(
    dateBetween(now(), prop("成交日期"), "years") >= 1,
    "🎂 第 " + format(dateBetween(now(), prop("成交日期"), "years")) + " 年 — " +
    if(
      dateBetween(now(), prop("成交日期"), "months") % 12 <= 1,
      "🎉 周年快到了！",
      ""
    ),
    format(dateBetween(now(), prop("成交日期"), "days")) + " 天"
  )
)
```

#### 下次關懷日（Formula）

**欄位名稱**：`下次關懷日`
**公式**：

```
if(
  not(prop("Day0 感謝")),
  prop("成交日期"),
  if(
    not(prop("Day7 入住關心")),
    dateAdd(prop("成交日期"), 7, "days"),
    if(
      not(prop("Day30 轉介紹")),
      dateAdd(prop("成交日期"), 30, "days"),
      if(
        not(prop("Day90 二次轉介紹")),
        dateAdd(prop("成交日期"), 90, "days"),
        dateAdd(prop("成交日期"), 365, "days")
      )
    )
  )
)
```

**說明**：根據目前完成到哪個節點，自動計算下一個應關懷的日期。4 個節點都完成後，設定為成交周年日。

   - 新增欄位 → 類型選 Formula → 貼上公式
   - 先用簡單版測試，確認可以運作後再換進階版
6. **建立視圖**（按照 1.4）：
   - 點左上角的「+ Add a view」
   - 選擇 Board / Calendar 等視圖類型

### Step 6：建立 Relation 關聯（約 10 分鐘）

**這步最重要！讓 5 個資料庫互相連結。**

## 常見問題 FAQ

### Q1：免費版 Notion 夠用嗎？

**A**：一個人使用完全夠用。免費版限制：
- Block 數量無上限（2024 年後已移除限制）
- 檔案上傳每個限 5MB
- 可建立的資料庫數量無限

**建議**：先用免費版，等到團隊超過 2 人再升級到 Plus（每月 $10 美元）。

### Q2：公式貼上去出錯怎麼辦？

**A**：常見原因和解法：
1. **屬性名稱不一致**：公式裡的 `prop("最後互動日")` 必須和你的欄位名稱「完全一樣」，包括空格
2. **屬性類型錯誤**：Formula 裡用到的屬性，類型要正確（例如 dateBetween 需要 Date 類型）
3. **先用簡單版**：建議先貼簡單版公式確認可以運作，再換進階版

### Q3：手機上可以用嗎？

**A**：可以。Notion 有 iOS 和 Android App。
- Board 視圖在手機上要左右滑動
- 建議手機上主要用 Table 視圖和 Calendar 視圖
- 新增記錄在手機上操作很方便

### Q4：可以多人協作嗎？

**A**：可以。
- 免費版可以邀請最多 10 位 Guest
- 用 Person 屬性分配「負責人」
- 每個人可以建立自己的視圖和篩選器
- 團隊模式建議升級到 Plus 方案

### Q5：資料可以匯出嗎？

**A**：可以。
- Notion 支援匯出為 CSV（適合匯入 Excel）
- 也可以匯出為 Markdown
- 設定 → Export → 選擇格式

### Q6：跟其他工具（如 Google Calendar）可以同步嗎？

**A**：可以透過以下方式：
- **n8n 自動化**：用課程提供的 n8n 工作流（見 `templates/n8n/`）
- **Notion 日曆功能**：Notion Calendar 可以和 Google Calendar 同步
- **Zapier / Make**：第三方自動化工具

### Q7：我已經有用其他 CRM 系統了，可以同時用嗎？

**A**：建議不要。
- 資料分散在多個系統，反而更混亂
- 建議選定一個系統後全部移過來
- 如果現有 CRM 有匯出功能，可以匯出 CSV 再匯入 Notion
